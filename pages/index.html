About Flakes

Just think about how easy it has become to build a web application. Think about all the frameworks and tools you have available to get started. When you’re building for the web, someone has already done the hard work. Mostly, it is about finding the right parts and putting them together. However, this abundance of tools and frameworks is lacking when it comes to internal applications.

Flakes is a front-end framework that helps businesses build better internal software. I very rarely come across internal applications that have been well designed and developed. Usually, time constraints and business priorities get in the way. The user experience is almost always an afterthought. Mobile readiness non-existent. And an aesthetic that is as old as the internet itself.

Flakes is different. Well designed from the ground up. An uncluttered interface that lets you get work done without jarring your sensibilities. Responsive so you can manage your business on the go. Built around a component architecture for easy development. And much more.

The Flakes aesthetic is sharp, uncluttered, task driven and not influenced by todays trends. Thousands of hours are spent on internal applications which makes speed a priority. Flakes was designed to be weightless. It goes away with custom fonts, images or slow loading libraries. All you are left with is a blazingly fast framework.

Framework Overview

Flakes is boring

It is built on top of layers of abstractions and loosely coupled code to make building application front-ends simpler. It avoids using vain effects, styles or images that do not contribute to the function of the application. The framework does not assume to know how it will be used. That complicated mumbo jumbo just to say flakes takes care of all the boring stuff so you can launch your project faster.

Built for the real world

When designing flakes, I took the time to study plenty of real world internal applications. CRM software, Inventory Management, Website Admins, Transaction Management, etc. I wanted Flakes to be agnostic enough that any one of those applications could be built with it.

Component Architecture

Flakes is a curated set of components that are most useful in building internal applications. This modular architecture allows you to swap out pieces of Flakes for more appropriate components. It also makes it very simple for contributors to add their own components.
